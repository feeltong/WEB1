overflow 의  scroll , hidden 등은 내부 콘텐츠가 넘쳤을 떄 반응하는 반응형 문법?
float 의 left 와 right 를 통해 요소를 배치 시키며 clear 의 both 등을 통해 해제 할 수있다.

overflow 속성

overflow 속성은 박스 모델 안에서 , 안의 콘텐츠가 박스를 벗어날 떄 어떻게 처리할 지를 결정한다
예를 들어 , 박스의 높이를 벗어난 텍스트가 있을 떄 , 스크롤을 보여줄 것인지 또는 벗어난 영역을 숨길 것인지
등을 결정할 수 있다.

visible :  기본 값 , 박스를 벗어나는 콘텐츠가 그대로 보임
scroll : 우측에 스크롤 바를 노출하여 , 스크롤 바를 통해 콘텐츠 확인 가능 (평상시에 많이 쓰는 것인듯!)
hidden : 벗어나는 콘텐츠를 모두 숨겨버립니다 (이걸로 진우한테 짤 보내야겠다 메모)
auto : scroll 속성 ㄱ밧은 스크롤 바가 항상 노출 되지만 , auto 속성 값은 콘텐츠가 박스를 벗아날 떄에만 스클롤 바를 노출합니다.
inherit : 부모의 속성 값을 물려 받습니다 (상속?)

flaot 속성은 이름 그대로 요소를 띄워서 왼쪽이나 오른쪾에 배치할 수 있도록 만들어 준다
왼쪽이나 오른쪽으로 float 속성을 적용한 요소의 다음 요소들 안의 텍스트들은 적용한 요소 옆에 위치하게 됩니다.

none : 기본 속성으로 요소를 띄우지 않는다
left : 요소를 왼쪽으로 띄웁니다
right : 요소를 오른쪽을 띄웁니다
inherit : 부모의 속성을 (상속!)받습니다.

이 속성은 특히 레이아웃을 잡을 때 많이 사용하는 속성이면서 , 초보자들이 제일 어려워하는 속성 중에
하나다 ! 제너릭 보다는 덜 어렵다 개인적으론 ㅠ

clear 속성 

이 속성은 float 를 해제시킴으로써 줄 바꿈 되어 구역을 새롭게 시작한다 이 속성은 float 의 left 와 right 를 
선택해서 해제할 수 있고 둘다 동시에 해제할 수도 있다

none : 아무것도 안한다!
left : float left 를 해제한다
right : float right 를 해제한다
both : 다 해제한다 
inherit : 말 아낌
